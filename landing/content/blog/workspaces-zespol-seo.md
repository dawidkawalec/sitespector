## Wprowadzenie: dlaczego workspace'y mają znaczenie

PLACEHOLDER: Content agent — Hook dla agencji i zespołów: wiele stron, wielu klientów — potrzeba oddzielenia i kontroli dostępu. SiteSpector workspace'y = logiczna separacja projektów. 2 zdania.

## Czym jest workspace w SiteSpector

PLACEHOLDER: Content agent — Definicja: kontener na projekty (strony), audyty, raporty. Jeden workspace = np. jeden klient lub jedna marka. Krótki opis. Kto tworzy, kto zarządza.

## Role: Owner, Admin, Member

PLACEHOLDER: Content agent — Opisz każdą rolę: Owner (pełna kontrola, billing), Admin (zarządzanie członkami, projektami), Member (dostęp do audytów, raportów — bez zarządzania). Tabela lub bullet points z uprawnieniami.

## Scenariusz 1: agencja z wieloma klientami

PLACEHOLDER: Content agent — Jeden workspace na klienta. Owner/Admin zaprasza specjalistów SEO jako Members. Klient widzi tylko swój workspace (jeśli ma dostęp). 2-3 zdania z praktycznymi wskazówkami.

## Scenariusz 2: freelancer z kilkoma projektami

PLACEHOLDER: Content agent — Workspace per projekt lub per marka. Jeden użytkownik = Owner. Prosty przypadek. Kiedy warto tworzyć osobne vs łączyć.

## Scenariusz 3: dział SEO w firmie

PLACEHOLDER: Content agent — Jeden workspace „Firma”, kilku Members. Admin zarządza projektami (strony produktowe, landing, blog). Wspólne raporty. Krótko.

## Jak ustawić workspace w praktyce

PLACEHOLDER: Content agent — Krok po kroku: tworzenie workspace, zapraszanie członków, przypisywanie ról. Odwołanie do dokumentacji/panelu. CTA do wypróbowania.

## Podsumowanie: dobre praktyki

PLACEHOLDER: Content agent — 3-4 wskazówki: oddzielaj klientów, używaj minimalnych uprawnień, regularnie przeglądaj członków. CTA do SiteSpector.
