Praktyczne snippet-y JSON-LD (gotowe do adaptacji)

Przykłady poniżej są wzorcami startowymi. Zastąp domenę, nazwy, adresy, numery telefonów i linki wartościami Twojej firmy. Po wdrożeniu koniecznie wykonaj walidację (Rich Results Test + Search Console).

1) Organization / LocalBusiness

{% raw %}
{
  "@context": "https://schema.org",
  "@type": "Organization",
  "name": "Nazwa Firmy",
  "url": "https://twojadomena.pl",
  "logo": "https://twojadomena.pl/logo.png",
  "sameAs": [
    "https://www.facebook.com/twojprofil",
    "https://www.linkedin.com/company/twojprofil"
  ],
  "contactPoint": [{
    "@type": "ContactPoint",
    "telephone": "+48-123-456-789",
    "contactType": "customer service",
    "areaServed": "PL"
  }]
}
{% endraw %}

2) WebSite + SearchAction

{% raw %}
{
  "@context": "https://schema.org",
  "@type": "WebSite",
  "url": "https://twojadomena.pl/",

  "potentialAction": {
    "@type": "SearchAction",
    "target": "https://twojadomena.pl/szukaj?q={search_term_string}",
    "query-input": "required name=search_term_string"
  }
}
{% endraw %}

Jak wdrożyć i jak sprawdzić po wdrożeniu

Krok Co zrobić Jak zweryfikować
1. Wdrożenie JSON-LD Wstaw JSON-LD w `<script type="application/ld+json">` w `<head>` lub na końcu `<body>` odpowiednich podstron. Sprawdź source strony i upewnij się, że kod jest renderowany w finalnym HTML.
2. Uzupełnienie pól wymaganych Dla każdego typu uzupełnij pola required (name/url/offers itd.) oraz rekomendowane. Usuń wszystkie „Braki wymagane” w tej sekcji raportu.
3. Test Rich Results Przetestuj URL-e krytyczne narzędziem Google Rich Results Test. Brak błędów krytycznych i ostrzeżeń wysokiego priorytetu.
4. Search Console Po indeksacji monitoruj raporty „Ulepszenia” i „Strony”. Wzrost prawidłowych elementów structured data i brak nowych błędów.
5. Monitoring biznesowy Obserwuj CTR, TOP10 i sekcje URL z wdrożonym Schema. Wzrost CTR i większa stabilność widoczności na frazach transakcyjnych.
